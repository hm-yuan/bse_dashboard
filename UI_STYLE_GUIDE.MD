# UI_STYLE_GUIDE.md

## 1. 设计定位

本项目是“交易所画像 Dashboard”，用于对外展示和内部演示。

当前 UI 方向调整为：

> 现代蓝色主基调 + 瑞士国际主义数据分析风格。

本项目不再以传统 Shiny 页面、通用 SaaS 卡片或炫酷大屏作为视觉目标，而是将交易所画像系统做成一种更克制、更清晰、更有版式秩序的金融数据产品界面。

目标气质：

* 现代
* 简洁
* 干净
* 专业
* 高级
* 克制
* 信息秩序强
* 有金融数据产品感
* 有交易所画像系统的正式感

视觉锚点：

> 瑞士国际主义版式秩序 + 现代蓝色金融 Dashboard + 北京证券交易所品牌识别。

`guizang-ppt-skill` 中的瑞士国际主义风格用于提供设计方法论：网格优先、强字体层级、单一锚点色、hairline 细线、非对称排版和信息优先。当前项目是 Shiny Dashboard，不是横向翻页网页 PPT，因此不套用该 skill 的 S01-S22 页面模板锁，也不使用 WebGL/ASCII 背景和演示动效要求。

## 2. 设计系统取舍

### 2.1 Swiss International Typographic Style：主视觉方向

瑞士国际主义风格是当前 UI 的主导方向。

借鉴内容：

* 严格网格和清晰对齐
* 强字号对比和明确字重阶梯
* 左对齐为主的信息组织
* 非对称布局和明确视觉主次
* 1px hairline 分割线
* 极少装饰，依赖排版、留白和结构建立高级感
* 单一蓝色锚点，高亮服务于数据判断

使用场景：

* 页面头部
* KPI 区
* 主图区域
* 洞察区
* 明细表
* 风险标签
* 导航和筛选控件

设计目标：

> 让页面像一张秩序清晰的瑞士风数据分析海报，同时具备现代金融 dashboard 的可交互能力。

### 2.2 BSE Brand：品牌识别来源

BSE 品牌仍是项目的身份来源，但品牌色必须被纳入瑞士风的克制配色系统。

使用规则：

* 顶部导航左侧使用 `www/logo_bse.png` 官方 LOGO。
* LOGO 保持原始比例，不拉伸、不裁切、不变形。
* 页面主锚点色使用深蓝和科技蓝。
* BSE 红只用于风险、异常或少量品牌点缀。
* BSE 青只用于流动性、交易活跃等辅助语义，不参与大面积装饰。

### 2.3 Carbon Edge：结构线和数据表达

Carbon Edge 继续作为结构线、图表和表格规范的参考，但视觉层级要向瑞士风收敛。

借鉴内容：

* 清晰坐标轴、图例和网格线
* 表格高信息密度
* 风险色克制使用
* 卡片标题区和内容区分割
* 数据视图的边界和层级

### 2.4 Ant Design Pro：页面组织

Ant Design Pro 只作为中后台页面组织参考，不作为视觉风格来源。

借鉴内容：

* KPI 区
* 筛选区
* 明细表
* 页面分区
* 可扩展业务结构

禁止把页面做成普通 Ant Design 模板感。页面应更接近“瑞士风数据分析版式”，而不是“后台管理系统模板”。

## 3. 核心设计原则

### 3.1 网格优先

所有页面都必须服从统一网格。

规则：

1. 页面主体采用 12 或 16 列网格。
2. 页面左边界、标题、KPI、主图和明细区应对齐到同一内容轴。
3. 卡片之间的间距使用 8px 体系。
4. 不用随意 margin 调位置；优先使用 grid gap、列宽和行高。
5. 页面不必平均分布，应允许主图更大、辅助信息更窄，形成主次关系。

### 3.2 字体层级建立秩序

瑞士风的高级感主要来自字号、字重、行距和留白。

规则：

1. 页面标题必须明显大于卡片标题。
2. KPI 数字必须明显大于 KPI 名称。
3. 小字不能过细，避免投屏和截图时不可读。
4. 标题左对齐，不使用居中大标题作为常规页面样式。
5. 中文标题优先短句，不用长标题硬塞。

### 3.3 单一蓝色锚点

页面的主高亮色应只有一个蓝色锚点。

规则：

1. 深蓝用于标题、导航、页面编号和核心文本。
2. 科技蓝用于本所高亮、当前页激活、核心数据和重要线条。
3. 绿色、橙色、红色只用于状态和风险语义。
4. 不使用多种高饱和色做装饰。
5. 不使用大面积渐变、炫光、粒子和毛玻璃。

### 3.4 Hairline 是主要结构工具

使用 1px 细线表达模块边界。

规则：

1. 卡片边框使用 1px 浅蓝灰。
2. 标题区和内容区使用 hairline 分割。
3. 表格使用细线和强对齐，不使用重色块。
4. 图表网格线弱化，只保留阅读所需辅助线。
5. 不使用厚重阴影表达层级。

### 3.5 信息优先

所有视觉元素必须服务数据阅读和对外讲述。

禁止：

* 为装饰添加无意义图形。
* 为统一而牺牲页面主次。
* 让表格或辅助图抢主图视觉权重。
* 用大面积风险红制造紧张感。
* 把页面做成指标堆砌大屏。

## 4. 色彩系统

### 4.1 设计 Token

CSS 应集中定义以下 token，并优先在 `www/custom.css` 中使用：

```css
:root {
  --swiss-navy: #002B5B;
  --swiss-blue: #1D6FEA;
  --swiss-bg: #F5F8FC;
  --swiss-card: #FFFFFF;
  --swiss-line: #D8E2F0;
  --swiss-text: #0F172A;
  --swiss-muted: #64748B;
  --swiss-green: #16A34A;
  --swiss-orange: #F59E0B;
  --swiss-red: #D92D20;
}
```

### 4.2 主色

| 颜色 | 色值 | 用途 |
| --- | --- | --- |
| 深蓝主色 | `#002B5B` | 页面标题、导航、页面编号、核心文本 |
| 科技蓝强调 | `#1D6FEA` | 本所高亮、当前页面、核心数据、关键线条 |
| 浅蓝背景 | `#F5F8FC` | 页面背景 |
| 白色卡片 | `#FFFFFF` | KPI、图表、洞察、明细卡片 |
| 细线边框 | `#D8E2F0` | hairline、卡片边框、表格分割线 |

### 4.3 文本和状态色

| 颜色 | 色值 | 用途 |
| --- | --- | --- |
| 主文字 | `#0F172A` | 正文、图表标签、表格正文 |
| 次级文字 | `#64748B` | KPI 名称、说明、注释、数据来源 |
| 正向绿色 | `#16A34A` | 正向变化、改善、达标 |
| 提示橙色 | `#F59E0B` | 中度风险、提示、关注 |
| 风险红色 | `#D92D20` | 高风险、异常、预警 |

### 4.4 使用规则

1. 蓝色是唯一主锚点色。
2. 本所、本市场、当前选中对象统一使用科技蓝。
3. 对标市场使用低饱和蓝灰。
4. 绿色只用于正向变化。
5. 橙色和红色只用于风险、异常和提示。
6. 不使用大面积渐变。
7. 不使用多色装饰条。
8. 不使用透明毛玻璃和强阴影。

## 5. 字体系统

### 5.1 字体选择

优先使用系统无衬线字体：

```css
font-family:
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  "Helvetica Neue",
  Arial,
  "Noto Sans SC",
  "Microsoft YaHei UI",
  "Microsoft YaHei",
  sans-serif;
```

数字、代码、口径编号可使用等宽字体：

```css
font-family:
  "SFMono-Regular",
  "Cascadia Mono",
  "Consolas",
  "JetBrains Mono",
  monospace;
```

### 5.2 字号层级

| 层级 | 用途 | 建议 |
| --- | --- | --- |
| 页面编号 | 视觉锚点 | 48-72px，600-700，深蓝或浅蓝灰 |
| 页面标题 | 页面主题 | 32-42px，700，深蓝或近黑 |
| 页面副标题 / 核心判断 | 判断说明 | 16-20px，500 |
| 区块标题 | 卡片和内容区标题 | 16-20px，600 |
| KPI 数字 | 核心指标 | 28-40px，700 |
| KPI 名称 | 指标标签 | 12-14px，600，灰蓝 |
| 正文说明 | 洞察和解释 | 13-15px，400-500 |
| 注释 / 口径 | 数据来源、说明 | 11-12px，500，低饱和灰 |

### 5.3 字重规则

瑞士风在 Dashboard 中不机械套用“超大字 200”的演示页规则。由于本项目是中文数据产品，标题和 KPI 需要保持投屏可读。

规则：

1. 页面标题和 KPI 数字使用 700。
2. 区块标题使用 600。
3. 正文使用 400-500。
4. 12-14px 小字不低于 500。
5. 不使用过细字重作为正文。
6. 页面级大编号可使用 600，不要比标题更抢。

### 5.4 文案规则

1. 页面标题短而直接。
2. 主图标题使用判断型标题，不只写指标名称。
3. 每个页面必须有一句话核心判断。
4. 洞察说明控制在 3-5 条。
5. 避免大段文字。
6. 指标名称保持一致。

## 6. 网格、间距和画布

### 6.1 基础间距

采用 8px 间距体系：

| Token | 值 |
| --- | --- |
| `space-1` | 4px |
| `space-2` | 8px |
| `space-3` | 12px |
| `space-4` | 16px |
| `space-5` | 20px |
| `space-6` | 24px |
| `space-8` | 32px |
| `space-10` | 40px |
| `space-12` | 48px |
| `space-16` | 64px |

### 6.2 页面画布

建议：

* 页面背景：`#F5F8FC`
* 页面外边距：24-32px
* 页面最大内容宽度：不强制居中窄版，优先适配 1280-1600px 宽屏。
* 卡片间距：16px 或 20px。
* 主图区和页面标题之间保留更大呼吸空间。
* 明细表区可以信息密度更高，但必须保持列对齐。

### 6.3 响应式原则

第一版优先保证桌面端展示效果。

推荐适配：

* 主要设计宽度：1440px 及以上。
* 最小可接受宽度：1280px。
* 对外展示建议：16:9 屏幕。

## 7. 顶部导航规范

顶部导航采用白色或极浅色商务导航，并使用北京证券交易所官方 LOGO。

推荐方向：

```css
height: 64px;
background: #FFFFFF;
border-bottom: 1px solid var(--swiss-line);
box-shadow: none;
```

规则：

1. 导航栏高度控制在 64px 左右。
2. 底部使用 1px hairline 分割。
3. 左侧使用 `www/logo_bse.png`。
4. LOGO 高度建议 36-42px，宽度自适应。
5. 导航菜单文字使用深蓝或灰蓝，字号约 14px，字重 600。
6. 当前页面激活态使用深蓝文字 + 底部 2px 科技蓝线。
7. 不使用圆润 pill、透明气泡、毛玻璃或大面积背景块。
8. hover 状态只加深文字或显示 1px 线条。

## 8. 卡片系统

### 8.1 卡片类型

| 卡片类型 | 用途 |
| --- | --- |
| Swiss Page Header | 页面编号、标题、核心判断 |
| KPI Strip | 顶部核心指标横向条 |
| KPI Card | 单个指标 |
| Chart Card | 主图容器 |
| Insight Card | 关键洞察 |
| Detail Card | 明细表和钻取 |
| Status Card | 风险、项目、处置状态 |
| Filter Card | 筛选条件 |

### 8.2 卡片样式

统一规范：

```css
border-radius: 6px;
background: var(--swiss-card);
border: 1px solid var(--swiss-line);
box-shadow: none;
```

规则：

1. 圆角控制在 4-8px。
2. 优先使用细边框和留白表达层级。
3. 阴影极轻或不用阴影。
4. 卡片标题区和内容区使用 1px hairline 分割。
5. 图表卡片、KPI 卡片、洞察卡片、明细卡片必须风格统一。
6. 不使用嵌套卡片。
7. 不使用大面积渐变卡片。

### 8.3 Swiss Page Header

页面头部用于建立瑞士风版式锚点。

建议元素：

* 页面编号，如 `01`、`02`、`03`、`04`。
* 页面标题。
* 一句话核心判断。
* 可选数据口径说明。

规则：

1. 页面编号可作为大号视觉锚点，但不能抢 KPI 和主图。
2. 标题和判断左对齐。
3. 头部底部可使用 1px hairline。
4. 不使用 hero 大色块作为常规页面头。

### 8.4 KPI Strip

KPI 区应从“普通卡片堆叠”调整为克制的横向指标条。

规则：

1. 每页 KPI 数量控制在 5-6 个。
2. KPI Strip 使用统一网格等宽或轻微非对称列宽。
3. KPI 项之间用 1px hairline 分隔。
4. KPI 数字使用深蓝或科技蓝。
5. KPI 名称使用灰蓝小字。
6. 变化值使用 `metric_delta()`，不使用大面积彩色背景。
7. 风险页 KPI 可使用橙/红标签，但必须克制。

### 8.5 Chart Card

Chart Card 包含：

* 判断型图表标题
* 口径提示
* 图表主体
* 简洁图例
* 必要标注

规则：

1. 主图优先使用大卡片和更大空间。
2. 不要在一个 Chart Card 中堆叠过多图形。
3. 坐标轴、图例、网格线统一。
4. 标题区和图表区之间使用 hairline。
5. 图表背景白色或极浅灰。
6. 重点标签清晰可读。

### 8.6 Insight Card

Insight Card 用于 3-5 条关键发现。

规则：

1. 洞察解释主图，不重复 KPI。
2. 每条洞察包含编号、标题和一句说明。
3. 编号使用直角小方块或等宽数字，不使用圆形胶囊。
4. 洞察项之间使用 hairline 或浅灰分隔。
5. 文案应可直接用于对外汇报。

### 8.7 Detail Card

Detail Card 用于明细钻取。

规则：

1. 表格高信息密度但强对齐。
2. 表头使用细线和轻底色，不使用重色块。
3. 数字列右对齐。
4. 文本列左对齐。
5. 风险、状态、变化使用统一标签。
6. 行 hover 保持轻量。

## 9. 图表系统

### 9.1 图表总体规则

1. 图表服务页面核心问题。
2. 每页保留 1-2 个核心主图或组合主图区域。
3. 辅助图用于解释，不抢主图视觉。
4. 图例放在图表上方或右上角。
5. 坐标轴和网格线使用浅灰。
6. 本所数据使用科技蓝高亮。
7. 对标市场使用低饱和蓝灰。
8. 风险图表使用浅橙到红色。
9. 不使用 3D、厚阴影、复杂渐变。
10. 图表标题使用判断型标题。

### 9.2 气泡图和散点图

适用页面：

* 市场定位
* 公司画像
* 市场质量画像

规则：

* 本所或重点对象使用科技蓝。
* 对标对象使用低饱和蓝灰。
* 气泡大小必须有明确含义。
* 坐标轴说明清楚。
* 重点标签不遮挡数据。
* 使用少量标注解释象限或关键位置。

### 9.3 矩阵和热力图

适用页面：

* 公司画像
* 市场质量画像

规则：

* 行列对齐清楚。
* 低值浅色，高值深色。
* 经营贡献矩阵使用蓝色阶。
* 风险热力图使用浅橙到红色阶。
* 图例简洁，不使用过饱和色块。

### 9.4 趋势图

适用页面：

* 市场生态
* 市场定位中的成交趋势

规则：

* 趋势线突出方向。
* 同一图中折线不超过 3 条。
* 柱线组合表达数量和金额。
* 关键事件使用轻量标记。
* 时间轴标签不拥挤。

### 9.5 地图

适用页面：

* 公司画像

规则：

* 地图是公司地域分布证据，不是装饰背景。
* 省级色块低饱和。
* 城市气泡使用蓝色体系。
* 点击态和筛选态必须清楚。
* 地图下方或右侧联动内容应对齐到同一网格。

### 9.6 表格

表格适用于明细钻取，不作为主视觉。

规则：

* 表头浅底或白底 + hairline。
* 行高保持紧凑但可读。
* 使用轻量斑马纹或仅用行分割线。
* 数字列右对齐。
* 风险等级使用统一 `risk_tag()`。
* 表格默认显示重点列，更多字段通过展开行或后续二级页面承接。

3. 成交趋势、行业结构、企业性质作为辅助证据区。
4. 图表网格允许非对称，主图跨更多列。
5. 本所数据使用深蓝或科技蓝突出。

页面重点表达：

> 本所在多层次资本市场中的位置、体量、活跃度、估值和结构特征。

### 10.3 公司画像

页面定位：

瑞士风布局方向：

页面重点表达：

## 11. 图标、状态和风险标签

### 11.1 图标风格

图标应极简、线性、低装饰。

规则：

* 优先使用现有图标库或简洁线性图标。
* 不使用复杂拟物图标。
* 不混用多套风格。
* 图标不能成为主要装饰。
* 瑞士风 UI 中可用直角小方块、编号和 hairline 替代部分图标。

### 11.2 风险标签

统一风险等级：

| 风险等级 | 样式 |
| --- | --- |
| 低风险 | 浅绿底 + 绿色文字 |
| 中风险 | 浅橙底 + 橙色文字 |
| 高风险 | 浅红底 + 红色文字 |

规则：

1. 风险标签短而明确。
2. 不使用大圆角胶囊，圆角控制在 4-6px。
3. 同一风险等级全项目样式一致。
4. 风险解释放在洞察区或明细表中。

### 11.3 变化值

`metric_delta()` 用于表达同比、环比或状态变化。

建议：

* 正向：绿色文字或浅绿背景。
* 负向：红色文字或浅红背景。
* 中性：灰蓝文字。
* 提示：橙色文字或浅橙背景。

## 13. 禁止事项

1. 禁止做成传统 Shiny 默认控件风格。
2. 禁止做成炫酷大屏。
3. 禁止所有页面使用完全相同布局。
4. 禁止每页堆叠过多图表。
5. 禁止过度使用红色。
6. 禁止多色高饱和装饰。
7. 禁止大面积渐变、发光、粒子背景。
8. 禁止毛玻璃和厚重阴影。
9. 禁止卡片圆角过大。
10. 禁止在 R 文件中大量写内联 CSS。
11. 禁止主图和辅助图视觉权重不分。
12. 禁止表格占据主视觉。
13. 禁止指标名称前后不一致。
14. 禁止导航栏使用圆润 pill、透明气泡或半透明块作为当前页面激活态。
15. 禁止为了瑞士风而破坏数据可读性。

## 14. Codex 执行要求

当 Codex 开发或修改 UI 时，应遵循：

1. 开始 UI 修改前先阅读本文件。
2. `UI_STYLE_GUIDE.MD` 是本项目唯一视觉规范文件，不新增 `DESIGN_TARGET.md`，除非用户明确要求。
3. 页面视觉以“现代蓝色主基调 + 瑞士国际主义数据分析风格”为准。
4. 瑞士风原则转译为 Shiny Dashboard 网格、字体、hairline、低装饰和单一蓝色锚点。
5. 所有新增样式优先写入 `www/custom.css`。
6. 所有通用 UI 组件优先写入 `R/ui_components.R`。
7. 不要破坏既有模块化结构。
8. 不要改动数据读取逻辑、指标计算口径、processed 数据结构和 Wind 数据维护逻辑。
9. 必要时只微调 `chart_*.R` 的图表样式。
10. 修改完成后检查明显运行错误。

## 15. 验收标准

视觉验收：

1. 页面整体不再像传统 Shiny 默认界面。
2. 页面具备瑞士国际主义特征：强网格、强字体层级、细线分隔、克制蓝色主调、信息秩序清楚。
3. 页面背景干净，没有复杂装饰。
4. KPI 区比普通卡片更克制、更有设计感。
5. 图表和表格具备强对齐和清晰层级。
6. 风险色使用克制，不刺眼。
7. 四个画像页面视觉语言统一，但布局性格不同。

工程验收：

1. CSS 主要集中在 `www/custom.css`。
2. 通用 UI 组件主要集中在 `R/ui_components.R`。
3. 页面模块仍保持模块化。
4. 数据读取、metrics 和 chart 核心逻辑没有被破坏。
5. `shiny::runApp()` 可以正常启动。

最终目标：

> 一个现代、简洁、高级、克制、强网格、强信息秩序的蓝色瑞士国际主义金融数据 Dashboard。
