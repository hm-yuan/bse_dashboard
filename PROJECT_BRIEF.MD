# PROJECT_BRIEF.md

## 项目背景

本项目建设一个“交易所画像 Dashboard”，用于对外展示和内部演示。

项目目标是用可视化方式讲清楚北京证券交易所在资本市场中的定位、上市公司结构、市场发展情况和市场质量风险。

当前版本重点是形成可演示的 R Shiny 原型：页面要能讲故事，图表要能支撑判断，数据层要能逐步从演示数据切换到真实 processed 数据。

## 总体叙事

Dashboard 的叙事主线是：

看定位 -> 看公司 -> 看发展 -> 看质量。

即：

1. 市场定位：本所在资本市场中处于什么位置？
2. 公司画像：本所服务的是一批什么样的公司？
3. 市场生态：本所业务功能是否持续增强？
4. 市场质量画像：本所运行是否健康、风险是否可控？

`config/page_blocks.yml` 中保留“首页总览”页面模型，但当前 `app.R` 只挂载四个画像页。首页后续可作为汇报开场页恢复，不影响当前四页演示主线。

## 当前实现概览

当前 App 已实现：

* 四个画像页面导航：市场定位、公司画像、市场生态、市场质量画像。
* 现代蓝色主基调 + 瑞士国际主义数据分析风格视觉系统。
* `semantic` / `placeholder` 两套页面模型。
* processed 标准表读取、演示表补齐、基础真实数据处理和质量日志。
* Highcharter 主图体系、reactable 公司明细表和若干前端辅助脚本。

当前数据状态：

* `data/raw/` 中有上市公司基本情况、市场板块成交统计、周度日均成交、北交所日度成交、Wind 市场交易统计、全球主要资本市场、时间序列和北证 A 股走势等原始输入。
* `data/processed/` 中有市场定位 KPI、公司明细、城市经纬度表和九张标准演示表。
* `data/cache/` 中有中国地图和城市坐标缓存。

## 页面一：市场定位

核心问题：

本所在资本市场中处于什么位置？

当前页面定位：

市场定位页是当前演示中信息最密集的页面，围绕规模、成交、估值、上市节奏、企业性质和行业结构展开。

已实现图表和组件：

* 多市场规模气泡图：横轴总市值，纵轴日均成交额，气泡大小表示上市公司数量。
* 北交所上市公司数量与交易成长：历年上市数量 + 交易规模成长，支持日度/月度/年度和成交额/换手率/成交占比切换。
* 估值、股权性质与市值分布：市值与市盈率散点图 + 企业所有权性质分布 + 上市公司市值区间分布。
* 全球主要资本市场对比：支持总市值、2026 年日均成交额、2026 年日均换手率、市盈率 TTM 切换，并可筛选中国市场和成长板块。
* 层层递进的市场结构：各市场规模对比 + 各板块年度成交对比。
* 服务创新型中小企业：各市场行业分布 + 北交所国家级专精特新企业数量 + 研发强度 > 5% 企业占比。
* 行业矩形树图能力仍保留，支持按公司家数/市值切换，并可下钻查看公司。

辅助 KPI：

* 上市公司数量
* 总市值
* 流通市值
* 日均成交额
* 市盈率中位数
* 前十大公司市值占比

页面讲述重点：

本所相对于其他市场是什么体量、什么活跃度、什么估值水平、什么产业结构，并进一步说明北交所企业的上市节奏、所有权结构和市值分布特征。

维护边界：

新增图表必须服务“市场定位”叙事。不要继续把所有市场相关图都放入本页，能进入明细或后续二级页面的内容不应挤占主图区域。

## 页面二：公司画像

核心问题：

本所上市公司是什么样的企业？

当前页面定位：

公司画像页已从早期“行业矩阵 + 四象限”演进为地理分布、经营表现、经营状态、行业结构和公司明细联动的画像页面。页面模型仍保留行业矩阵和四象限配置，但当前模块实际渲染以地图联动卡和底部行业结构组件为主。

已实现图表和组件：

* 中国省级公司分布地图：省级色块表示上市公司数量，城市气泡表示城市公司数量。
* 地图点击筛选：点击城市或省份后联动下方散点图、经营状态环图和右侧公司表，并支持复位。
* 营业收入 vs 净利润散点图：展示公司经营规模和利润分布。
* 公司经营状态环图：展示筛选范围内经营状态分布。
* reactable 公司明细表：支持搜索、分页、展开详情。
* 展开详情：展示公司基本情况、经营业绩和近 30 日股价走势。
* 各市场行业分布图和行业矩形树图：用于补充行业结构观察。

辅助 KPI：

* 营业收入中位数
* 净利润中位数
* 盈利公司占比
* ROE 中位数
* 研发费用率中位数
* 募集资金使用比例

页面讲述重点：

本所公司主要分布在哪些地区和行业，经营规模和盈利状态如何，创新属性和专精特新特征是否突出，哪些公司或行业构成当前画像的主要样本。

维护边界：

公司画像不是单家公司档案系统。公司级展开详情服务于解释群体画像，不应让页面转向复杂 CRM 或个股分析工具。

## 页面三：市场生态

核心问题：

本所的业务功能是否在持续增强？

当前页面定位：

市场生态页当前主要由页面模型统一渲染，是时间序列叙事页面。重点观察上市、融资、交易活跃和生态功能的阶段变化。

主图：

* 上市与融资发展趋势：新增上市柱状图 + IPO / 再融资金额折线。
* 交易活跃与市场生态趋势：成交金额、活跃公司、换手率或生态指标的多轴组合。

辅助 KPI：

* 本年新增上市公司
* 在审企业数量
* IPO 融资额
* 再融资额
* 年度成交金额
* 活跃公司数量

页面讲述重点：

本所发展动能来自哪里，是上市扩容、融资深化、交易活跃提升，还是产品和服务生态完善。

后续优先方向：

* 增加关键事件标注。
* 增加融资类型和审核状态筛选。
* 补充上市审核、融资结构和产品生态明细。

## 页面四：市场质量画像

核心问题：

本所市场运行是否健康，风险是否可控？

当前页面定位：

市场质量页当前主要由页面模型统一渲染，是风险驾驶舱型页面。重点观察流动性、基本面质量、估值、财务、合规和退市风险。

主图：

* 市场质量状态矩阵：横轴流动性，纵轴基本面质量，气泡大小表示市值，颜色表示风险等级。
* 风险类型-行业热力图：颜色越深代表该风险类型在行业中的集中度越高。

辅助 KPI：

* 低流动性公司占比
* 高估值低增长公司数量
* 连续亏损公司数量
* 监管措施数量
* 募投异常公司数量
* 退市风险公司数量

页面讲述重点：

市场整体质量如何，风险集中在哪些行业、哪些类型、哪些公司，哪些对象需要持续跟踪。

后续优先方向：

* 将风险阈值集中维护在 `config/thresholds.yml`。
* 强化风险公司明细和处置状态。
* 增加风险类型筛选、行业筛选和风险变化趋势。

## 数据策略

当前采用“真实基础数据 + 标准演示表 + 语义页面模型”的过渡策略。

标准表：

* `dim_company`
* `dim_industry`
* `fact_market_period`
* `fact_financial_period`
* `fact_financing`
* `fact_fundraising_use`
* `fact_supervision`
* `fact_risk_tag`
* `fact_listing_pipeline`

辅助表：

* `market_position_kpi`
* `market_position_company_detail`
* `city_coordinates`
* `data_quality_log`

数据原则：

1. 原始数据放在 `data/raw/`，应用运行时不覆盖、不改写。
2. 页面优先通过 `R/data_access.R` 消费 `data/processed/`。
3. 当前市场定位页部分图表函数直接只读 raw Excel，用于全球市场对比、北交所交易成长、行业/市值/PE/企业性质等图表；公司画像中部分 raw Excel 读取用于联动筛选、财务表和展开详情。这些都属于过渡实现，后续应迁移到 processed 标准表。
4. 所有演示生成字段必须在数据文档中标注为演示字段，不能当作真实业务事实。
